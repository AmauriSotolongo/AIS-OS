# Knowledge Compounding

El principio de que un knowledge base se vuelve exponencialmente más valioso a medida que crece, porque las conexiones entre piezas de conocimiento crecen más rápido que el conocimiento mismo.

## En el contexto del LLM Wiki

Ver [[llm-wiki-pattern|LLM Wiki Pattern]]. La diferencia clave entre RAG clásico y un wiki LLM: el RAG re-deriva desde cero en cada consulta; el wiki acumula. Cada fuente que se ingesta no solo agrega una página — actualiza todas las páginas relacionadas. El valor crece de forma compuesta.

## Analogía con código

El wiki es la codebase. El LLM es el programador. Amauri es el product owner. Como un codebase bien mantenido, el wiki mejora con cada PR — no solo se agranda.

## Aplicación para Amauri

[[1Klick]] y [[Digital Compass]] generan conocimiento constantemente: calls con clientes, propuestas, reuniones, research de mercado. Sin un sistema de compounding, ese conocimiento se pierde en notas dispersas o en la memoria. Con este wiki, cada fuente que Amauri agrega hace que el sistema sea más útil para la siguiente pregunta.
